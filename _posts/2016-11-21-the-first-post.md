This is the first blog post of mind using [Jekyll](https://jekyllrb.com/) and
[Poole](https://github.com/poole/poole) on
[Github Pages](https://pages.github.com/).

This blog is created with these three reasons in mind.

First, my notes of the command and knowledge that I have learned are usually all
over the places, so I want to, at least, gather them together in one place.
I usually keep the command I learned in Google Keep. It is a good place in terms of
accessibility, and searching, but I found myself forgetting the context of the command.
Thus this blog should force me to at least provide context for the command I learned.
It may fail to archive that, but that should be fine too :). In addition to Google Keep,
I also put my note onto the code that I was writing. This one seems to have a better
context, but I have a hard time searching through code base to find just one or two line
of code that I want. I do not put these note in Google Keep as they heavily rely
on the context of the code. Thus I hope this blog should gather all my notes to be in one
place.

Secondly, writing seems to require a lot of preparation, and well-ordered thought, so I hope
this blog will my thought well ordered. I usually bad at explaining my idea. I am also
experiencing the situation where I have less opinion on the topic. Sometimes, my idea
is not well-ordered enough for the writing task, and it is causing me problems.
So trying to write something out should improve the organization of my thought.

The last reason is that this blog should be the way that I can practice writing more.
It is true that the English class require a lot of writing, but I usually do not
happy to do that, so this blog which is going to be a lot more technical
should force me to practice writing more.

The content of this blog will be about the knowledge that I encounter. It may include some
event in life that I want to note, but it should be mainly about the command, knowledge,
framework, algorithm, and etc.

I will not cover the process that I took to set this blog up since
[Joshua Lande](http://joshualande.com/about/) already did a great job on
describing the steps. You can see his blog post
[here](http://joshualande.com/jekyll-github-pages-poole).
